# Codex Personal Guidelines

## Skill Naming Conventions

Use consistent naming patterns to make skills easier to reference and discuss.

- Use gerund form (verb + -ing).
- Use lowercase letters, numbers, and hyphens only.
- Avoid vague or generic names (e.g., helper, utils, tools, documents, data, files).
- Avoid reserved words (e.g., anthropic-helper, claude-tools).
- Keep naming consistent across the skill collection.

### Good examples

- processing-pdfs
- analyzing-spreadsheets
- managing-databases
- testing-code
- writing-documentation

### Acceptable alternatives

- pdf-processing
- spreadsheet-analysis
- process-pdfs
- analyze-spreadsheets
